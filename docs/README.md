# Venopai Documentation Guide

This directory contains the canonical documentation for the Venopai ecommerce platform. Files are organized by purpose to keep onboarding, operations, and audits clean and maintainable.

## Structure
- architecture/ : System design, core flows, and technical architecture notes.
- operations/ : Runbooks, incident response, observability, backup, and launch operations.
- deployment/ : Deployment topology and release runbooks.
- security/ : Security controls, compliance, and hardening guidance.
- testing/ : Testing strategy, load testing, and validation checklists.
- product/ : Product requirements, roadmap, and design system documentation.
- onboarding/ : Developer onboarding and environment setup.
- reference/ : API contract, schemas, and AI context notes.
- archive/ : Historical or superseded documents retained for traceability.

## Recommended Onboarding Flow
1. onboarding/ENGINEERING_ONBOARDING.md
2. onboarding/ENVIRONMENT_SETUP.md
3. reference/API_CONTRACT.md
4. architecture/CART_ARCHITECTURE_ANALYSIS.md
5. deployment/DEPLOYMENT_OVERVIEW.md
6. operations/OPERATIONS_INDEX.md

## Operations and Deployment
- Deployment steps live in deployment/.
- On-call, incident, and observability guidance live in operations/.
- Security checklists and compliance live in security/.

## Archive Policy
Archived files in archive/ are retained for context but should not be treated as current guidance unless explicitly referenced.
